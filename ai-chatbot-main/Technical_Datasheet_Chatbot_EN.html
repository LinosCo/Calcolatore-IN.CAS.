1. INTRODUCTION

The IN.CAS. AI Chatbot is an advanced WordPress plugin that integrates OpenAI's artificial intelligence to provide an intelligent and contextual virtual assistant on the corporate website. Specifically designed for the medical sector and safety innovation, the chatbot offers real-time support to visitors, answers technical questions, and can capture qualified leads.

OBJECTIVE: Provide an automated and personalized support experience based on artificial intelligence, improving visitor engagement and optimizing the management of information requests about the S.P.R.I.Z. system and IN.CAS. products.

4. INSTALLATION GUIDE

Method 1: Installation via WordPress Admin (Recommended)

1 Plugin Download

Download the plugin ZIP file voler-ai-chatbot.zip provided by the IN.CAS. technical team.

2 WordPress Panel Access

In the WordPress admin panel, navigate to Plugins > Add New.

3 File Upload

Click Upload Plugin at the top, select the ZIP file and click Install Now.

4 Activation

After installation, click Activate Plugin. The plugin will automatically create the necessary database tables.

POST-INSTALLATION CHECK: After activation, verify that "Voler AI Chatbot" or "AI Chatbot" appears in the WordPress sidebar menu.

Method 2: Installation via FTP

  1. Extract the ZIP file contents into a folder on your computer
  2. Connect via FTP to the WordPress server
  3. Upload the extracted folder to wp-content/plugins/
  4. Go to Plugins > Installed Plugins and activate the plugin

6. KNOWLEDGE BASE CONFIGURATION (RAG)

The RAG (Retrieval-Augmented Generation) system allows the chatbot to access specific information about the IN.CAS. site.

Enable Knowledge Base

1 Activate RAG System

In the "Knowledge Base (RAG) Settings" section, check "Enable Knowledge Base".

2 Content Type Selection

Select post types to index:

  • ‚úì Pages (static site pages)
  • ‚úì Posts (blog articles)
  • ‚úì Products (if WooCommerce is present)
3 Upload Technical Documents

Upload IN.CAS. informational documents:

  • Product technical sheets (.pdf)
  • User manuals (.pdf)
  • FAQs and guides (.md, .txt)
  • Certifications and regulations (.pdf)
  • Case studies (.md, .pdf)
SUPPORTED FORMATS:
  • .txt - Plain text files
  • .md / .markdown - Markdown files
  • .pdf - PDF documents (with automatic text extraction)
4 Content Reindexing

After uploading documents or updating WordPress content:

  1. Click "Reindex Content"
  2. Wait for process completion (may take several minutes)
  3. Verify in "Snapshot Info" section that a new snapshot was generated
IMPORTANT NOTE: Reindexing generates embeddings via OpenAI API, consuming tokens. Plan reindexing only when necessary (new documents, significant updates).

Similarity Threshold Configuration

The Similarity Threshold determines how "similar" retrieved chunks must be to be considered relevant:

Value Behavior When to use
0.7 - 0.8 Standard - Balanced Recommended default configuration
0.8 - 0.9 Strict - High precision When avoiding imprecise answers is important
0.6 - 0.7 Permissive - More flexible To retrieve more correlated information

IN.CAS. Recommendation: Set to 0.75 to balance precision and coverage.

Regular Maintenance

Review Analytics Weekly Analyze engagement metrics and lead quality
Export Leads Weekly Export leads and import to corporate CRM
Clean Chat History Quarterly Delete history over 90 days (GDPR)

OpenAI Cost Optimization

COST-SAVING STRATEGIES:
  • Use higher similarity threshold (0.8) to reduce irrelevant chunks
  • Limit Assistant Instructions length (fewer tokens per prompt)
  • Set appropriate message threshold for lead form (avoid overly long conversations)
  • Monitor analytics to identify inefficient conversations
  • Use more economical GPT models if sufficient (e.g., GPT-3.5 vs GPT-4)

Content Best Practices

  • Knowledge Base Documents: Keep files under 5MB, prefer .md format for better parsing
  • Assistant Instructions: Be specific but concise, focus on medical sector expertise
  • Suggested Questions: Update regularly based on most frequently received questions
  • Welcome Message: Clearly communicate chatbot purpose and capabilities
